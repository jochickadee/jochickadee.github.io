Ruby Intro – Arrays and Hashes

December, 7th 2015

Warning: this blog contains painfully brutal honesty and self-reflection along with tech talk. Cheers!

Everythings moving along great. Then BOOM! I'm totally lost. But we'll get to that. Onto the real reason for this blog post - arrays and hashes.

Arrays and hashses are two different ways of storing data in Ruby that you can access later in your program. Arrays are an ordered list of values (or items). For example here's a list of my pets in an array: 


	    		pet_array = ["Stupido", "Sarone", "Maximus", "Jörmungandr"]
See, just a list of my pets names. But there is an order to this list. Stupido is first so to access him you would use the index 0 (in programming, you start with 0 rather than 1), Sarone is next to he's number 1, Maximus is number 3 and so on down the line. Now hashes are a little different. Hashes store information in pairs. So for this example I would have what kind of animal each pet is and then the name with it. Example: 

	    		pet_hash = {
	    		cat => "Stupido",
	    		ball_python1 => "Sarone",
	    		boa_constrictor => "Maximus",
	    		ball_python2 => "Jörmungandr"
	    		}
With a hash you can get different information based on what you ask. "Hey hash, how many pets do I have?" "4"; "Hey hash, what's my boa constrictors name?" "Maximus" and so on.
