Weeks Two and Three

HTML & CSS!

November, 29th 2015

Weeks two and three in the bag! I feel like I'm getting into the swing of Phase 0. The last two weeks we've been learning HTML and CSS. HTML is like the skeleton of a website and CSS is like the skin. With CSS you can style the HTML elements and position them in whatever layout you design. I'm going to go over positioning in CSS because frankly, I totally didn't get it! Quick CSS lesson - you use the selector to say which HTML element you'd like to style and then the declaration code block to tell the browser how to style that element.

Positioning in CSS

You can use the position property to position elements on a page. To do this you set the position property and then give direction coordinates (Top, Bottom, Left and Right) to move your element. The position property takes four different values: static, relative, fixed, and absolute. This determines how the element will move on the page. Let's start with the easy peasy one: Static.

Static is your default position. It's how the page renders with no positioning set. It's just the way the HTML is naturally. You can't give it any directional coordinates. It just is.

Next up is fixed. Fixed is good for when you want to have an element stay where it is in the browser window regardless of scrolling. (Like say a navigation bar at the top of your page.) You can use directional positioning properties on a fixed element and it will just float over any content that runs behind it. So as you scroll down your page your fixed navigation bar will just float over the text and images on your page. Fixed position elements move in relation to the browser window. If you give your nav bar a top property of 50px, it will move 50px from the top of the browser window.

Warning: The next two are kind of a pain in the butt!

Relative. When you set your directional properties on a relative element, it will move away from it's original position. So if you give your image a top: 50px declaration it will move down 50px from it's original position, not 50px down from the top of the browser window. Also, there will be an empty space where that element used to be.

Absolute positioning. Elements with absolute positioning move relative to the nearest positioned parent element, except static elements. Example time! So if you have a div that's positioned relative and your image is inside of it when you give it an absolute position and a 50px top property, it will move 50px down from the relative div. If there is no positioned parent element it will move relative to the browswer window. It also can float over other content like fixed position elements can.

Well, that's positioning in a nutshell. Thanks for reading thus far. Hopefully I'm not completely wrong in my understanding of positioning. If I am, feel free to shoot me a lengthy email informing me of my wrongness.
